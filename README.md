# kakebo — シンプル家計簿アプリ

> 「給与は決まっている。カードで使いすぎない。それだけ。」

ブラウザだけで動く、一人用の家計簿 Web アプリです。React + TypeScript + Vite で構築され、データは端末の `localStorage` に保存されます。GitHub Pages に静的サイトとして配信されます。

---

## 1. プロジェクト概要

**コンセプト**：今月あといくら使えるかを、開いた瞬間に把握できることに特化した家計簿。

**想定ユーザー**

- 一人暮らし / 独身
- 月収がほぼ固定（給与制）
- クレジットカードをメインの支払い手段にしている
- 細かい集計より「予算残高の見える化」と「使いすぎアラート」が欲しい

**設計思想**

- ダッシュボードを玄関にする（残り予算が最初に目に入る）
- 入力は数タップで完結
- アラートは色（緑 / アンバー / 赤）で直感的に伝える

---

## 2. 機能一覧

アプリは以下の 7 画面で構成されます。

### ダッシュボード（`/`）

- 今月の残り予算を大きく表示（円）
- 残り日数 / 今日の上限金額（残り予算 ÷ 残り日数）を表示
- 全体プログレスバー：80% 超でアンバー、100% 超で赤
- カテゴリ別の消化率をプログレスバーと「注意 / 超過」バッジで表示
- 画面表示時に固定費の自動計上をチェック（月初の初回アクセスで当月分を一括登録）

### 支出入力（`/add`）

- レシート撮影 / 画像選択 → Claude API（Vision）が金額・日付・店名を読み取りフォームへ自動セット
- 手動入力：金額・カテゴリ・メモ・日付
- レシート OCR には設定画面で保存した Anthropic API キーが必要

### 取引一覧（`/transactions`）

- 月セレクタで対象月を切り替え
- 日付の降順で当月の取引を表示
- 各行から削除可能

### 予算（`/budget`）

- カテゴリの追加・編集・削除
- カテゴリ名・月の上限金額・色を設定
- カテゴリ合計を画面上部に表示
- 初期カテゴリ：食費 / 交通費 / 日用品 / 娯楽・交際 / 被服 / その他

### 固定費（`/fixed`）

- 家賃・サブスク・通信費などを名前・金額・カテゴリ・計上日（1〜28 日）で登録
- 毎月初回のダッシュボードアクセス時に当月分の支出として一括登録
- 同月内の二重計上を防ぐためのフラグ付き（`kakebo_last_fixed_applied`）

### インポート（`/import`）

- クレカ・銀行の明細 CSV をアップロード
- 列名から金額（`金額` / `amount` / `利用額` / `出金`）・日付（`日付` / `date` / `利用日` / `年月日`）・店名（`店名` / `内容` / `摘要` / `memo` / `description` / `利用先`）を自動検出
- カテゴリ名と店名のキーワード一致でカテゴリを仮割当（マッチしないものは「その他」）
- プレビュー画面でチェックを外したり、行ごとにカテゴリを修正してから一括取り込み

### 設定（`/settings`）

- Anthropic API キーの保存（パスワード表示の切り替え可）
- 予算・カテゴリ設定 / 固定費管理へのリンク
- データエクスポート：JSON または CSV ファイルとしてダウンロード

## 3. セットアップ手順

### 必要環境

- Node.js 20 以上
- npm

### 手順

```bash
# クローン
git clone https://github.com/<your-account>/kakebo.git
cd kakebo

# 依存パッケージをインストール
npm install

# 環境変数ファイルを作成（任意：レシート OCR を使う場合）
cp .env.example .env   # 存在しない場合は新規作成
# .env に以下を記述
# VITE_ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxx

# 開発サーバ起動
npm run dev
```

ブラウザで `http://localhost:5173/kakebo/` を開きます（`vite.config.ts` で `base: '/kakebo/'` を設定しているため）。

> 補足：Anthropic API キーは `.env` に書かなくても、アプリの設定画面から登録できます。設定画面で入れたキーは `localStorage` に保存され、レシート読み取り時に使われます。

### ビルド / プレビュー

```bash
npm run build      # tsc + vite build
npm run preview    # ビルド済み成果物のローカル確認
```

## 4. 使い方（典型的な月次フロー）

1. **予算を設定する**：`/budget` でカテゴリごとの月の上限金額を決める。初期カテゴリをそのまま使ってもよいし、追加・削除・色変更も可能。
2. **固定費を登録する**：`/fixed` で家賃・サブスクなどを登録。毎月、初回のダッシュボード表示時に当月分が自動計上される。
3. **日々の支出を記録する**：以下のいずれか好きな方法で。
   - **手動入力**：`/add` で金額・カテゴリ・メモ・日付を入力。
   - **レシート OCR**：`/add` でレシート画像を撮影 / 選択 → AI が金額・日付・店名を抽出 → 内容を確認して保存。
   - **CSV インポート**：`/import` でクレカ明細 CSV を取り込み、プレビューでカテゴリを微調整して一括登録。
4. **ダッシュボードで残額確認**：`/` を開くたびに「今月の残り」「今日の上限」「カテゴリ別の消化率」を確認。
5. **必要に応じてバックアップ**：`/settings` から JSON / CSV でエクスポート。

## 5. デプロイ

GitHub Pages への自動デプロイが `.github/workflows/deploy.yml` に設定されています。

- `main` ブランチへ push するだけで GitHub Actions がビルド → Pages へ公開
- `vite.config.ts` の `base` がリポジトリ名（`/kakebo/`）と一致している必要あり
- リポジトリの **Settings → Pages** で「Build and deployment」のソースを **GitHub Actions** に設定しておくこと

公開 URL の例：`https://<your-account>.github.io/kakebo/`

## 6. データの扱い

- すべてのデータはブラウザの **`localStorage`** にのみ保存されます。サーバへの送信はありません（Anthropic API へのレシート画像送信を除く）。
- 使用している `localStorage` キー：
  - `kakebo_categories` — カテゴリ
  - `kakebo_transactions` — 取引
  - `kakebo_fixed_costs` — 固定費
  - `kakebo_settings` — 設定（API キーを含む）
  - `kakebo_last_fixed_applied` — 固定費を最後に自動計上した月（`YYYY-MM`）
- バックアップは `/settings` → 「データエクスポート」から JSON または CSV として取得できます。
- インポート機能（CSV）は明細取り込み専用で、エクスポートした JSON のリストア機能はありません。バックアップとして JSON を保管しつつ、必要時はブラウザ DevTools から `localStorage` に直接書き戻す形になります。

---

## 7. 注意点

- **API キーの保存場所**：Anthropic API キー（`sk-ant-...`）は **ブラウザの `localStorage` に平文で保存** されます。共有 PC では使用しないでください。設定画面でキーを空にして保存すれば削除できます。
- **ブラウザから直接 Anthropic API を呼ぶ**：レシート OCR は `anthropic-dangerous-direct-browser-calls: true` ヘッダーを付けてブラウザから直接呼んでいます。CORS 上で動作しますが、API キーがネットワークログ等から漏れないよう自分専用の環境で利用してください。
- **複数端末同期はできません**：データは端末ごとの `localStorage` に閉じています。別端末で使いたい場合は JSON エクスポート → 手動で取り込みが必要です。
- **ブラウザのストレージを消すとデータも消えます**：シークレットモード、ストレージクリア、別ブラウザ利用などでデータは引き継がれません。定期的なエクスポートを推奨します。
- **CSV の列構成**：金額と日付の列が検出できないファイルはエラーになります。列名に「金額 / amount / 利用額 / 出金」「日付 / date / 利用日 / 年月日」のいずれかが含まれている必要があります。
- **固定費の計上日**：1〜28 日のみ指定できます（月末日のばらつきを避けるため）。

## v0.4 シリーズで追加された主要機能

Phase 1 のピボット（家計簿→キャッシュフロー破綻早期警告）の実装が進み、以下の機能が追加されました。

### ホーム（Dashboard）
- **給料日カード**：今日の残高・給料日までの引落・入金前残高・黒字/赤字判定（v0.4.13）
- **進行中サイクル**：給料日サイクル中のカード/非カード利用累計、カード別バー（v0.4.15-19）
- **給料日までの引落リスト**：確定済引落の一覧、件数・金額（v0.4.14）
- **同日着弾アラート**：複数カードが同じ日に引き落とされる際の警告

### 新画面
- **タイムライン**：給料日2サイクル分の利用日ドット・引落日ドット・請求期間バーを横軸で可視化（PC優先、スマホ縦は警告）
- **カード管理**：請求グループとカードを階層管理、カードマスタから一括追加

### CSV取込
- セゾン・イオン明細CSVを自動判別・取込（Shift_JIS対応、メタデータ抽出）
- 請求一括レコード自動生成＋重複制御
- インポート履歴

### データモデル拡張
- カードごとの**引落月オフセット**（当月/翌月/翌々月）に対応（v0.4.21）
- `actualWithdrawalDate` で実引落日を保持（請求遅延などの実態反映）

### 運用支援
- **テストモード**（v0.4.4）：ページロード毎に取引データをクリア
- **全データバックアップ/復元**（v0.4.3）：JSON 1ファイルで端末移行可能
- **バージョン表示**：右上に常時表示
- **pre-pushフック**（v0.4.22）：壊れたコードのpushを防止
