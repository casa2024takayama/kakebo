# AI協調ルール（Claude Code + Cursor）

このプロジェクトは **Claude Code** と **Cursor** を併用して実装する。  
以降の実装・修正では、以下のルールを共通前提とする。

## 1. 引落集計ロジックの統一
- Dashboard と Cashflow で、引落集計結果は常に一致させる。
- 同種の集計を別実装しない。既存の共通関数を使い回す。
- 仕様変更時は片方だけを直さず、共通関数を修正して両画面に反映する。

## 2. dedup 判定ルール（重複計上防止）
- 判定軸は **引落日基準** を原則とする。
- 優先順位:
  1. `actualWithdrawalDate`（実引落日）
  2. 理論引落日（カード締め/支払条件から派生）
- ただし互換性維持のため、必要に応じて「利用日ベースの請求期間カバレッジ」も併用してよい。

## 3. 返金（負数）データの扱い
- CSVの負数行は「支出のマイナス」ではなく、**収入（返金）** として登録する。
- `kind: 'income'` で保存し、金額は正数化する。
- メモは `[返金]` プレフィックスで識別しやすくする。

## 4. 変更時の運用ルール
- 日付計算・金額集計・状態変更（フラグ更新）は副作用が大きいため、1機能1責務で変更する。
- 同一症状の修正を複数箇所にコピペしない（将来の乖離を防ぐ）。
- バグ修正時は「どのデータソースを真実値として優先するか」をコミットメッセージに明記する。

## 5. レビュー観点（必須）
- 日付: 締め日・引落日・繰延・実引落日の優先順位が崩れていないか
- 金額: 符号（+/-）の意味が画面間で一致しているか
- 状態: `excludeFromWithdrawal` / `kind` 変更の副作用が他画面に波及しないか

## 6. Claude Code向け変更履歴（必須運用）
- 変更をコミットしたら、同じ内容を `docs/CLAUDE_CHANGELOG.md` に追記する。
- 追記フォーマットは「日時 / 担当AI / バージョン / 概要 / 影響範囲 / 検証結果」を最低限含める。
- Cursorで実装した変更も、Claude Codeが追跡できるよう同ファイルに統合記録する。
- 記録漏れを防ぐため、push前チェック項目として changelog 更新有無を確認する。
