# kakebo UI設計書（ピボット版）

**ピボット日:** 2026-05-02
**コンセプト:** カード中心の家計管理。「翌月引落予定 vs 収入 = 黒字 / 赤字」をトップ主役に。
**デザイン軸:** Apple Finance × ミニマル（Wallet / Apple Card Statement のメンタルモデル）

---

## 0. ピボットの本質

旧アプリは「カテゴリ別の使いすぎ防止」が主役だった。
新アプリは「**翌月口座から消える金額**」を主役にする。

理由：日本のクレカは「使った日」と「引き落とし日」がズレる。月内のカテゴリ消化率より、**「来月25日に何円ぶっ飛ぶか」**を見せる方が破綻防止に直結する。

### 1.2 新画面構成（5タブ + 設定階層）

```
/                       トップ（請求カレンダー＋黒字赤字）
/cards                  カード & 請求グループ管理
/add                    取引入力（カード選択必須）
/transactions           取引一覧（カード・グループでフィルタ）
/import                 銀行/カードCSV取込（固定費自動検出）
/settings               設定
  ├ /settings/groups    請求グループ（締め日・引落日）
  ├ /settings/loans     ローン（ボーナス月加算）
  ├ /settings/budget    カテゴリ予算（オプション）
  ├ /settings/rules     カテゴリ自動分類ルール
  └ /settings/data      エクスポート・インポート
```

### 1.3 新規追加されるドメイン概念

- **請求グループ (BillingGroup)**: PayPay / セゾン / イオン / JCB の4つ。締め日・引落日・引落口座を保持。
- **カード (Card)**: n枚。必ず1つの請求グループに所属（例：ニコス→JCBグループ）。
- **ローン (Loan)**: 月額＋ボーナス月加算（6月/12月など）。
- **引落予定 (BillingForecast)**: 月別に各グループの確定額を計算するビュー。

### 4.2 必須化の挙動仕様

- **デフォルト選択:** 直前の取引で使ったカードを記憶（`lastUsedCardId`）し、初期選択。ただし**ユーザーが明示的に選んだ扱い**にしない（保存ボタンは1タップで確認）。
- **未選択時のCTA文言:** 「保存（カードを選択）」とラベル変更。
- **エラー導線:** カード未登録の場合 → 「先にカードを追加 ▸」リンクで `/cards` へ。
- **現金もエンティティ化:** 「現金」も選択肢の1つ。請求グループに属さず、当月キャッシュフローに即計上。

## 6. デザイン原則

### 6.1 Apple Finance × ミニマル

| 原則 | 具体ルール |
|---|---|
| **数字は主役** | 最重要数字は 32–40px Bold。説明テキストは 12–14px Regular。 |
| **余白は富** | カード間 16px、セクション間 24px。詰め込まない。 |
| **角丸は16px** | カード・ボタン・モーダルすべて `rounded-2xl`。 |
| **影は最小** | `shadow-sm` のみ。フラットを基調。 |
| **カラーは3色まで** | 1画面に accent / signal / neutral 以外は出さない。 |
| **アニメーションは控えめ** | `transition-all duration-200`。バウンス禁止。 |

### 6.2 信号色の使い分けルール

| 色 | HEX | 意味 | 使う場面 | 使わない場面 |
|---|---|---|---|---|
| Green | `#1A6B4A` | 黒字・健全・確定済 | 黒字バッジ、保存ボタン、収入額 | 単なる強調 |
| Amber | `#E5972A` | 注意・予測・締め間近 | ギリギリ黒字、引落3日前、未確定金額 | 通常情報 |
| Red | `#C0392B` | 赤字・超過・警告 | 赤字バッジ、引落＞口座残、エラー | 通常の取消 |

**禁止事項:**
- 黒字なのに緑のプログレスバーを満タンにしない（消化率と黒字は別概念）
- 赤を「削除ボタン」と「赤字」両方に使わない → 削除は中性グレー＋確認モーダルへ

### 6.3 タイポグラフィ

```
Display    40px / 700  →  翌月引落予定の金額
Title      24px / 600  →  画面タイトル、セクションヘッダ
Body       16px / 400  →  本文、リスト項目
Caption    12px / 400  →  締め日、メタ情報
Mono Num   tabular-nums →  すべての金額（揃え用）
```

## 7. モバイルファースト原則（375px）

### 7.1 ファーストビュー（スクロールなし）に必ず置くもの

優先順位（上から）：

1. 月セレクタ（現在月 / 翌月切替）
2. **「翌月引落予定」金額**（最大サイズ）
3. **収入との差額＋黒字/赤字バッジ**
4. 直近1件の引落予定（日付・グループ・金額）

→ iPhone SE (375×667) で上記4要素までが画面内に収まること。

### 7.2 スクロール下層（ファースト→セカンド→サード）

```
[ファーストビュー]
  月セレクタ / 翌月引落 / 差額 / 直近1件
        ↓ スクロール
[セカンドビュー]
  引落カレンダー（月ビュー）
  グループ別内訳（4グループのプログレス＋金額）
        ↓ スクロール
[サードビュー]
  ローン（ボーナス加算月の警告含む）
  ＋脇役（カテゴリ別・今月の使用額）← 折りたたみデフォルト閉
```

### 7.3 タップ領域

- すべてのタップ要素は **最小 44×44px**（Apple HIG準拠）。
- カード選択チップは 80×56px 以上を推奨（指の太さ対策）。
- BottomNavのアイコンタップ領域は 64×56px。

### 7.4 横向き / タブレット

- 縦長レイアウトを基本とし、768px以上で2カラム化（左：カレンダー、右：グループ内訳）。
- 横向きスマホは縦レイアウトを維持（無理にレイアウト変更しない）。

## 8. アクセシビリティ：色だけに頼らない

### 8.1 信号色の三重符号化（色＋アイコン＋ラベル）

| 状態 | 色 | アイコン | テキストラベル |
|---|---|---|---|
| 黒字 | green ● | ✓ チェック | 「黒字」 |
| ギリギリ | amber ● | ⚠ 三角 | 「ギリギリ」 |
| 赤字 | red ● | ✕ バツ | 「赤字」 |

実装例（Tailwind）：

```tsx
<span className="inline-flex items-center gap-1 text-[#1A6B4A]">
  <CheckCircle aria-hidden className="w-4 h-4" />
  <span>黒字</span>
</span>
```

### 8.2 プログレスバー

- 80%超は amber、100%超は red にするが、**同時にパーセンテージ数値とアイコン**を併記する。
- 例: `███████░░ 82%  ⚠ 注意`

### 8.3 コントラスト

- テキスト/背景 4.5:1 以上（WCAG AA）。
- accent green `#1A6B4A` on white = 6.8:1 ✓
- amber `#E5972A` on white = 2.6:1 ✗ → **テキストには使わず背景バッジ専用**。文字色は白に。
- red `#C0392B` on white = 5.4:1 ✓

### 8.4 セマンティクスとARIA

- 金額には `aria-label="翌月引落予定 26万8400円"` を付与（スクリーンリーダー用、円記号を読み上げる）。
- カレンダーのグループアイコンは `aria-label="5月10日 PayPay引落 4万2300円"`。
- 必須項目の `*` は `aria-required="true"` と組み合わせる。

### 8.5 ダークモード

- 背景 `#0E0F11`、カード `#1C1D20`、テキスト `#F2F2F2`。
- accent green は `#3FAF85` に明度上げ（ダーク背景でのコントラスト確保）。
- amber/red も同様にトーンアップ版を用意。

### 8.6 動的フォントサイズ

- ルートに `font-size: 16px` を設定し、`rem` ベースで指定。
- ユーザーがOSのフォント拡大を有効にしてもレイアウト崩れない（折返しOK、横スクロール禁止）。
