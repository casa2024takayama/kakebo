## 1. デザイン哲学

### 1.1 「リッチさ」の定義 — なぜ薄いと言われたのか

現行実装の薄さの原因は3つある。

1. **情報の階層が立っていない**: ドット・バー・線・ラベルが同じ視覚プライオリティで並び、一目でどこを見るべきかが分からない。
2. **静的なベタ塗り**: 単色 fill ＋ box-shadow のみ。素材感がなく、紙やプラスチックの「物体感」が無い。
3. **時間の連続性が断絶**: 表示範囲の前後で情報がブツ切れ、未来も過去も「無いように見える」。長期表示前提では致命的。

### 1.2 リファレンス言語

| 参照 | 取り入れる語彙 |
|---|---|
| **Apple Wallet / Apple Card Statement** | カードカラーをそのままハイライトに使う、半透明ガラス（vibrancy）、丸みのある余白、`tabular-nums` の数字表示 |
| **Apple Health「アクティビティ」リング** | 円形要素のサイズで量を、色で種別を、二重符号化する文法 |
| **Mint / Monarch Money のキャッシュフロー** | 横軸時間 × 縦軸金額（or レーン）の単純構造に過去/未来のグラデーションで時制を与える |
| **Linear / Things のタイムライン** | 細い縦線・小さなラベル・密度の高い情報を破綻させない余白の取り方 |
| **iOS 17+ のフローティング HUD** | hover/tap で立ち上がるポップオーバーは半透明＋blur で「上に乗っている」感を出す |

### 1.3 情報密度と視認性のトレードオフ

- **目標密度**: 1サイクル（≒1ヶ月）あたり 30〜80件の取引が破綻なく可視化できる。
- **逃げ道**: 密度が閾値を超えたら自動的に「クラスタ」と「期間バー帯」に集約し、明細はポップオーバーへ。
- **守るべきもの**: ファーストビューで「今どこか／次の引落はいつ・いくらか」が0.5秒で読める。

## 2. 詳細レイアウト仕様

### 2.1 キャンバスとグリッド

PC（≥1280px）想定の基準キャンバスは以下。

```
canvas:        1200 × 520 px  (タイムラインカード本体)
outer padding: 24 px (上下) / 32 px (左右)
plot area:     1136 × 440 px  (canvas - padding)
```

横スクロール時は `min-width: 1136px` を保ちつつ、長期表示時は `1px ≒ 1日` を最小密度として伸長。**2年表示なら ≒ 730px の幅が最低でも必要** だが、可読性のため `5px/日` を推奨密度とし、横スクロールを許容する。

### 2.2 高さ配分（plot area 440px の内訳）

```
┌─────────────────────────────────────────────┐  ← top
│                                             │
│   上半分：利用日ドット                       │  170 px
│   ─ 同日クラスタ集約・カード色塗り           │
│                                             │
├─────────────────────────────────────────────┤  ← 170
│   中央軸：時間軸（目盛＋ラベル）              │  60 px
│   ─ 主目盛(月初)・補助目盛(週)・縦線          │
├─────────────────────────────────────────────┤  ← 230
│                                             │
│   下半分：請求期間バー → 引落日ドット         │  210 px
│   ─ レーン分け・接続線・バー右端→ドット      │
│                                             │
└─────────────────────────────────────────────┘  ← 440
```

**論点1**: 中央軸を 60px 確保するのは「月名ラベル＋日付目盛り＋今日タグ」を縦に重ねるため。社長判断で40pxに圧縮すれば情報をどれか落とす必要がある。

### 2.3 各要素の正確な px 仕様

| 要素 | 仕様 |
|---|---|
| 中央軸の主目盛線 | `border-top: 1px solid #1C1C1E33` (透明度 20%) |
| 主目盛(月初) tick | 縦 `8px` 下方に伸びる線、`1px solid #1C1C1E55` |
| 補助目盛(週・月曜) tick | 縦 `4px`、`1px solid #1C1C1E22` |
| 月ラベル | 12px / 600 / `#1C1C1E` 、月初の右3pxに baseline 配置 |
| 給料日縦線 | `2px solid #1A6B4A`、上下フル、`opacity: 0.7` |
| 給料日ラベル | 10px / 500 / 白文字 / 緑背景 `#1A6B4A` の chip、line上 8px、border-radius 4px |
| サイクル境界線 | `1px dashed #9CA3AF`、上下フル |
| **今日マーカー縦線** | `2px solid #C0392B`、上下フル、`box-shadow: 0 0 8px #C0392B66` で淡発光 |
| **今日タグ** | 10px / 700 / 白 / 赤背景 `#C0392B`、上端に「今日 5/3」、border-radius 6px、padding 2x6 |
| 利用日ドット直径 | 最小 `4px` 〜 最大 `20px`（金額レンジに応じて √スケール） |
| 引落日ドット直径 | 最小 `12px` 〜 最大 `36px`（金額レンジに応じて √スケール）+ `2px white ring` |
| 請求期間バー高さ | `10px`、border-radius `5px` |
| バー間レーン間隔 | `6px`（バー10 + gap6 = 16px ピッチ） |
| ドット → ラベル余白 | `4px` |
| フォント | `system-ui, "Hiragino Sans", "Noto Sans JP"`、数字は `tabular-nums` |

## 3. 時間軸の目盛り設計

### 3.1 目盛りヒエラルキー

```
        ┃                ┃                  ┃
        ┃ 給料日 5/15    ┃                  ┃ ← chip（緑）
        ┃                ┃ 今日 5/3         ┃
        ┃                ┃                  ┃ ← chip（赤・発光）
─┬──┬──╂──┬──┬──┬──┬──┬─╂┬──┬──┬──┬──┬──┬──╂──┬──→
 5月  │  │  │  │  │  │  │ 6月│  │  │  │  │  │  │
 主    補  補  補  補  補  主  補  補  補  補  補  主
```

- **主目盛**: 月初（毎月1日）に縦 8px tick + 月名ラベル「5月」を右肩に。
- **補助目盛**: 月曜の朝（週初）に縦 4px tick、ラベル無し。
- **日目盛**: 1日刻みの極細点線（`opacity: 0.05`）を背景に敷く。長期表示で密になりすぎる場合は ≥5px/日 のときのみ表示。

### 3.2 縦線（vertical guides）の3種

| 種別 | 色 | 太さ | スタイル | ラベル |
|---|---|---|---|---|
| **給料日** | `#1A6B4A`（accent green） | 2px solid | 不透明度70% | 緑 chip「給料日 M/D」を線上端 |
| **サイクル境界** | `#9CA3AF` | 1px dashed | 不透明度100% | ラベルなし（補助線） |
| **今日** | `#C0392B`（danger） | 2px solid | + glow | 赤 chip「今日 M/D」を線上端 |

**論点2**: 給料日のラベルを「給料日 5/15」とするか「給 5/15」と短縮するか。長期表示では文字密度が高くなる。推奨: ホバー時のみフルテキスト、平常時は丸アイコン「💴」+ 日付。

### 3.3 「今日」ラベルの選択肢

A案（採用推奨）: **タグ風 chip + 縦線** — 視認性最高、Apple Reminders の「今日」バッジと同じメンタルモデル。
B案: 吹き出し風（三角ポインタ付き） — Mint 風だが、密集すると邪魔。
C案: 縦線のみ＋アニメ pulsing dot — ミニマルだが「今日」のテキストが消えるため非推奨。

→ **A案で確定**。

## 4. 利用日ドット（上半分）

### 4.1 サイズマッピング

```
diameter = clamp(4, 4 + 16 * sqrt(amount / maxAmount), 20)
// maxAmount は表示範囲内の同種ドットの最大値
```

√スケールは「5,000円と50,000円」の見た目差を直線スケールより緩和し、極端な外れ値による潰れを防ぐ。

### 4.2 カードカラー適用ルール

- ドットの fill = カードカラー（§14のパレットから）
- 白縁 `2px solid white` を全ドットに付与（重なり時の輪郭確保）
- 過去（today より前）: opacity 100%
- 未来予定（手動入力された未来日取引）: opacity 60% + `border: 1.5px dashed`

### 4.3 同日集約（クラスタ）

- 同日 × 同カードは1ドットに合算（既存実装と同じ）
- 同日 × **複数カード** が並ぶ場合: 同じ x 座標から 6px ずつ垂直オフセット（最大4個まで縦に積む）
- 5個以上重なる場合: 最上位ドットの右上に `+N` バッジ（10px/700/白文字/グレー丸背景）

### 4.4 件数バッジ

- ドット直径 ≥ 12px: ドット中央に件数を白文字（9px / 700）
- ドット直径 < 12px: 右下に小さな丸バッジ（6px 直径、件数は表示せず点のみ）。ホバーで件数表示。

## 5. 請求期間バー（下半分）

### 5.1 形状

```
       │■■■■■■■■■■■■■■■■■■■■■■■■■■■│●
       └ 請求期間バー（角丸5px）       └ 引落日ドット
                                       (右端から細線で接続)
```

- 角丸: `border-radius: 5px`
- fill: カードカラーに線形グラデーション（左→右で `opacity 0.55 → 0.95`、未来感＝右が濃い）
- 縁: `1px solid` でカードカラーの hsl 明度 -10%
- 高さ: 10px

### 5.2 期間ラベル

- バー幅 ≥ 80px: バー内中央に白文字（10px / 600）「セゾン 4/11–5/10」
- バー幅 < 80px: バー外（直下4px）に小さく `12px / 500` グレー文字。
- バー幅 < 30px: ラベル非表示、ホバーでポップオーバーに表示。

### 5.3 重なり処理（レーン分け）

- カード単位でレーンを固定（カードのチェック順 = 上から下のレーン順）
- 1レーン = `16px`（バー10 + gap6）
- 同じカード内で複数の期間バーが時系列に並ぶ場合は同レーン

### 5.4 過去 / 現在 / 未来の視覚差

| 時制 | バー fill | 縁 | アニメ |
|---|---|---|---|
| 過去（end < today） | 通常グラデ + 全体 opacity 0.7 | 実線 | なし |
| 現在（start ≤ today ≤ end） | 通常グラデ + 微発光 `box-shadow: 0 0 6px cardColor55` | 実線、太め1.5px | 接続線が ≈2秒周期で薄く脈動 |
| 未来（start > today） | グラデ + opacity 0.85 | **破線 1px dashed** | なし |

### 5.5 継続性の表現（左右端のフェード／矢印）

長期表示で表示範囲外に期間が続く場合：

- **左端フェード**: plot area の左 24px に `linear-gradient(to right, white, transparent)` のオーバーレイ。バーが切れていることを示す。
- **左端 `‹` シェブロン**: 該当バーの左端に小さな `‹` アイコン（10px、カードカラー）を重ねる。「前期間あり」のシグナル。
- **右端も同様** に `›` で「次期間あり」を示す。
- スクロール可能であることを伝えるため、外枠右下に常時「⇄ ドラッグでスクロール」ヒント（hover 3秒後フェードアウト）。

## 6. 引落日ドット（下半分）

### 6.1 期間バーとの接続

```
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■▶───●
└ 期間バー右端                  └ 引落日ドット
   └ 1px solid 接続線（カードカラー、22%透明）
   └ 右端は小三角 ▶（4px）でドットへの「流れ」を示す
```

接続線の y 座標 = バー高さの中央。ドットの y 座標 = バーから 28px 下。

### 6.2 ドットサイズ

```
diameter = clamp(12, 12 + 24 * sqrt(amount / maxWithdrawAmount), 36)
```

利用日ドットより一回り大きく（最小12 vs 4）、引落の「重量感」を強調。

### 6.3 ラベルレイアウト

```
        ●●●  ← 引落日ドット（直径36px）
       ─────
       6/27(土→月)  ← 12px / 600 / カードカラー
       ¥158,200    ← 14px / 700 / #1C1C1E / tabular-nums
       セゾン       ← 10px / 500 / #6B7280
```

- 日付の括弧内に休業日シフト後の実引落日「6/27(土→月)」を表記。
- ラベルは中央揃え、ドット直下 4px から開始。
- 隣接ドットとラベルが重なる場合は片方を上方（ドット上 4px）に逃がす。

## 7. カードフィルタ UI

### 7.1 配置と構造

タイムライン本体の上、タイトルバーの直下に「フィルタバー」を新設。

```
┌─────────────────────────────────────────────────────────────────┐
│ タイムライン                       〈 ⟲ 〉  [集約 | 明細]        │
├─────────────────────────────────────────────────────────────────┤
│ 表示カード  [優先2枚] [全表示]                10/10件  ▾        │
│ ☑ セゾン  ☑ AEON  ☐ JCB  ☐ PayPay  ☐ 楽天  ☐ ニコス  ...      │
└─────────────────────────────────────────────────────────────────┘
```

### 7.2 チェックボックスデザイン

- 各カードチップ: `padding: 6px 10px`、`border-radius: 999px`(pill)、`border: 1.5px solid cardColor`
- 未選択時: 背景 white、文字グレー `#6B7280`、左にカードカラー丸 8px
- 選択時: 背景 cardColor 10%、文字 cardColor、左にカードカラー丸 8px + `✓`
- ホバー時: 背景 cardColor 5%
- カードカラーの順（§14 のデフォルト10色順）で左→右に並べる

### 7.3 プリセットボタン

| ボタン | 動作 |
|---|---|
| **優先2枚** | セゾン・AEON のみ ON、他 OFF（社長の主要2枚） |
| **全表示** | 登録カード全て ON |
| **クリア** | 全 OFF（タイムラインは「期間バーのみ」表示に縮退） |

プリセットボタンは小さく `12px / 600 / 白 / accent green pill` で左寄せ。

### 7.4 10枚超過時の UX

- 11枚目以降は「...」+ ▾ ボタンに格納し、ドロップダウンで追加選択。
- ON にできるのは合計10枚まで。11枚目を ON にしようとすると `tooltip: 「最大10枚まで表示できます。他のカードを外してください」` を出して操作不可。
- レーン数の物理上限を10レーンに固定（下半分 210px ÷ 16px/レーン ≈ 13、内3レーンを引落日ドット行に確保）。

### 7.5 フィルタ適用時の表示順

- 上→下: カード優先順位（社長設定の `displayOrder` フィールドで管理。Phase 1 では「セゾン > AEON > JCB > PayPay > その他」のハードコード）
- 同優先度内では `card.createdAt` 昇順

**論点3**: 「優先順位」を専用フィールドにするか、`/cards` 画面のドラッグ並び替えで管理するか。後者推奨だが Phase 1 のスコープ外なので一旦ハードコード。

## 8. インタラクション設計

### 8.1 ドット hover/tap ポップオーバー

```
                  ┌──────────────────────────┐
                  │ 5/2(金)  セゾンゴールド   │  ← 12px / 600
                  │ ¥12,400  3件              │  ← 16px / 700
                  │ ───────────────────────   │
                  │ ¥3,200 ローソン            │  ← 11px / 500
                  │ ¥6,800 Amazon             │
                  │ ¥2,400 ファミマ            │
                  └──────────────────────────┘
                           ▼              ← 三角ポインタ 8px
                          ●               ← 対象ドット
```

- 背景: `rgba(255,255,255,0.92)` + `backdrop-filter: blur(12px)` (vibrancy)
- 縁: `1px solid rgba(0,0,0,0.08)`
- 影: `0 8px 24px rgba(0,0,0,0.12)`
- 角丸: 12px
- パディング: 12px 14px
- 出現: `transform: translateY(4px) scale(0.96)` → `translateY(0) scale(1)` を 120ms `ease-out`

ポップオーバーの位置：
- 利用日ドット → ドット上方
- 引落日ドット → ドット右側（ドットが大きいため上下より横が見やすい）
- 画面端で見切れる場合は反対側に自動反転

### 8.2 期間バー hover

- バー全体 `filter: brightness(1.1)` + 縁 2px に太化
- 接続線と引落ドットも同時に `opacity: 1` で強調（普段0.85）
- ツールチップ: 「セゾンゴールド｜請求期間 4/11–5/10｜引落予定 6/4(月) ¥98,500｜29件」

### 8.3 サイクル前後ナビゲーション

| 操作 | 動作 |
|---|---|
| `〈` `〉` ボタン | 1サイクル分シフト（給料日サイクルに合わせて約30日） |
| `⟲` ボタン | 今日を含むサイクルにジャンプ |
| **横ドラッグ** | プロットエリア内の空白領域（ドット・バー以外）を pointerdown→drag で水平スクロール。慣性スクロール（150ms decay） |
| **トラックパッド水平スクロール** | ネイティブ scroll をそのまま使う |
| **キーボード** | `←` / `→` で1日、`Shift+←/→` で1週間、`Cmd/Ctrl+←/→` で1サイクル |
| **マウスホイール** | 縦スクロール = ページスクロール、`Shift+ホイール` = タイムライン水平スクロール |

### 8.4 集約 ⇔ 明細モード切替

- 集約モード（既定）: ドットは合算表示、ポップオーバーで明細を見る
- 明細モード: ポップオーバーが常時表示（ピン留め状態）。ドット数が少ない時のみ自動で全ピン。多い時は警告「明細50件超のため集約表示推奨」+ 自動リセット。
- トランジション: 各ドットのポップオーバーを順にフェードイン（stagger 20ms）。

## 9. 空白・密度コントロール

### 9.1 取引が少ない期間

- 連続して7日以上取引が無い区間は背景に縦縞ハッチ（`#0000000A` の45度ストライプ、4px間隔）を敷き「空白期間」を視覚化。
- 一切取引が無いサイクル全体: 中央に「この期間に取引はありません」グレー11px + イラストレス。

### 9.2 取引が密集する日のクラスタリング

- 1日に同カードで5件超: ドット中央に件数表示し、ポップオーバーは「上位5件 + 残りN件」表示。
- 1日に複数カードで合計10件超: その日付に縦の小さなアイコン「⚡ 高密度」（ホバーで詳細）。

## 10. アクセシビリティ

### 10.1 色だけに頼らない情報設計

- カードを色 + アイコン（カード名2文字略号「Se」「Ae」「JC」など）の二重符号化
- 過去/未来の区別は色透明度 + 縁の実線/破線で二重符号化
- 引落金額の大小は色強度ではなくサイズで表現（色は識別目的のみ）

### 10.2 スクリーンリーダー

- SVG/divルート要素に `role="img"` + `aria-label="タイムライン 2026年5月3日から2026年7月14日 引落予定3件 利用ドット42件"`
- 各ドットに `aria-label="5月2日 セゾンゴールド 12,400円 3件"`（既存実装の title をそのまま流用）
- フィルタチェックボックスは標準 `<input type="checkbox">` を非表示にして visually-hidden、視覚要素は装飾。スクリーンリーダーには「セゾン カード表示 オン/オフ」と読み上げ。
- タイムラインの下に常時「テーブル形式で表示」リンクを置き、押下で下部の簡易リスト（既存の「引落予定（時系列）」）にフォーカスを移動。

### 10.3 キーボードナビゲーション

| キー | 動作 |
|---|---|
| `Tab` | フィルタチップ → ナビボタン → タイムライン本体（roving tabindex） → 簡易リスト |
| タイムライン内 `←/→` | 隣接ドットへフォーカス移動（時系列順） |
| `Enter` / `Space` | フォーカス中のドットのポップオーバー開閉 |
| `Esc` | ポップオーバーを閉じる |
| `Home` / `End` | タイムラインの先頭/末尾へジャンプ |

## 12. モーション

### 12.1 初回読み込み

1. **0–200ms**: plot area の枠が `opacity 0 → 1` でフェードイン
2. **200–400ms**: 中央軸＋目盛りが左→右にワイプ（`clip-path: inset(0 100% 0 0)` → `inset(0 0 0 0)`）
3. **400–700ms**: 期間バーが左→右へ伸びるアニメ（`transform: scaleX(0)` → `scaleX(1)`、`transform-origin: left`）
4. **600–900ms**: ドットが下からポップ（stagger 15ms、`transform: translateY(8px) scale(0.6)` → `translateY(0) scale(1)` with overshoot 1.05→1.0）
5. **900ms 以降**: 今日マーカーが pulse 1回（scale 1.0→1.2→1.0）

すべて `cubic-bezier(0.22, 1, 0.36, 1)` (ease-out-quart) を採用。
`prefers-reduced-motion: reduce` のユーザーには全て無効化、即時表示。

### 12.2 フィルタ切替

- カードを OFF: 該当ドット・バーが `opacity 1 → 0` + `translateY(-4px)` (150ms)
- カードを ON: 逆方向 (200ms)、stagger 10ms

### 12.3 スクロール時

- 60fps を維持するため、ドット・バーの再描画は CSS transform のみで実施（reflow 禁止）
- 横スクロール時、今日マーカーが画面外に出たら **画面上端に小さな「今日 ↗」フローティングインジケータ** を表示（クリックで今日へジャンプ）。

## 13. HTML/CSS 表現案

### 13.1 SVG vs HTML/CSS の使い分け

| 部分 | 採用 | 理由 |
|---|---|---|
| 中央軸の線・tick・グリッド | **SVG** (`<line>`, `<g>`) | 細かい線描画は SVG が綺麗、ベクター拡大対応 |
| 期間バー | **HTML div + CSS gradient** | 角丸＋グラデーション＋アニメは CSS の方が手早い、レイヤーも軽い |
| ドット | **HTML div + border-radius:50%** | 件数バッジや子要素を入れやすい |
| 接続線（バー→ドット） | **SVG `<path>`** | 微妙な曲線・dash パターンが綺麗 |
| 今日マーカー縦線 | **HTML div + box-shadow** | glow 効果が CSS で楽 |
| 縦線（給料日・サイクル境界） | **HTML div** | 単純で十分 |
| ポップオーバー | **HTML div + backdrop-filter: blur** | vibrancy は CSS のみ |

### 13.2 グラデーション・blur・shadow の使い所（控えめに）

| 効果 | 適用箇所 | 強度 |
|---|---|---|
| 線形グラデーション | 期間バーの左→右 | opacity 0.55→0.95 |
| 放射グラデーション | 引落日ドット中央が明るく見えるよう微弱に | center white 0.1 → edge 0 |
| backdrop-filter blur | ポップオーバー、フィルタバーの上 | blur(12px) |
| box-shadow（影） | カード本体 `0 1px 3px rgba(0,0,0,0.06)`、ポップオーバー `0 8px 24px rgba(0,0,0,0.12)` | 控えめ |
| box-shadow（glow） | 今日マーカー縦線 `0 0 8px #C0392B66`、現在進行中の期間バー `0 0 6px cardColor55` | 弱め |

### 13.3 パフォーマンス考慮

- **仮想スクロール**: タイムラインは横方向なので必須ではないが、表示範囲外（off-screen）のドットは `display:none` で間引き。`IntersectionObserver` で監視。
- **render budget**: 一画面あたり同時レンダー要素は 300 個まで。それを超える場合は自動でクラスタリング強化。
- **再計算**: `useMemo` でドット・バー配列をキャッシュ。`transactions` か `cards` か `cycleOffset` か `filteredCardIds` の変化時のみ再計算。
- **アニメ**: GPU 加速のため `transform` / `opacity` のみアニメ、`top` / `left` / `width` を直接アニメしない。
