# システム仕様書（kakebo）

## 1. 文書メタ情報
- システム名: kakebo
- 種別: 一人用ブラウザ家計簿アプリ
- 目的: 「今月あといくら使えるか」を即時把握し、カード使いすぎを防ぐ
- 実行環境: モダンブラウザ（Chrome / Safari / Firefox 最新）
- 更新方針: 実装変更時に本書と `README.md` を同時更新

## 2. システム概要
- アーキテクチャ: フロントエンド単体（サーバレス）
- データ保存: ブラウザ `localStorage`
- 配信方式: GitHub Pages（静的配信）
- 主要技術:
  - React 18
  - TypeScript
  - Vite 5
  - Zustand
  - React Router
  - Tailwind CSS
  - Papa Parse
  - Anthropic API（レシート読取）

## 3. 画面一覧と責務
- `/` ダッシュボード
  - 給料日までの見込み残高、引落予定、利用進捗の可視化
- `/cashflow` キャッシュフロー
  - 収入・引落・日次見込みをベースにしたフロー確認
- `/timeline` タイムライン
  - カード利用日と引落日の時系列可視化
- `/cards` カード管理
  - 請求グループ、カード情報、締め・支払条件の管理
- `/add` 入力
  - 手動入力、請求一括入力、収入入力、レシート読取
- `/transactions` 取引一覧
  - 月単位の一覧確認と削除
- `/import` CSV取込
  - 明細CSVの取込、プレビュー、カテゴリ補正
- `/budget` 予算設定
  - カテゴリと予算額の管理
- `/fixed` 固定費管理
  - 定期支出の登録・編集・削除
- `/settings` 設定
  - APIキー、給与日設定、データエクスポート/復元

## 4. データモデル（概念）
- Category
  - 用途: 支出分類
  - 主属性: `id`, `name`, `monthlyBudget`, `color`
- Transaction
  - 用途: 収支レコード
  - 主属性: `id`, `kind`, `amount`, `categoryId`, `date`, `memo`
  - 補助属性: `actualWithdrawalDate`, `excludeFromWithdrawal` など
- FixedCost
  - 用途: 定期支出
  - 主属性: `id`, `name`, `amount`, `categoryId`, `dayOfMonth`
- BillingGroup
  - 用途: カード請求単位
  - 主属性: 締め日、支払日、繰延条件
- Card
  - 用途: カード個体
  - 主属性: `id`, `name`, `billingGroupId`, 利用状態
- Settings
  - 用途: アプリ全体設定
  - 主属性: APIキー、給与日、その他表示・運用設定

## 5. 永続化仕様（localStorage）
- 使用キー（主要）:
  - `kakebo_categories`
  - `kakebo_transactions`
  - `kakebo_fixed_costs`
  - `kakebo_settings`
  - `kakebo_last_fixed_applied`
- 互換性:
  - 起動時にマイグレーションを適用し、旧データ構造を救済
- バックアップ:
  - `settings` から JSON / CSV エクスポート
  - JSON復元機能あり（実装状態に依存）

## 6. 業務ルール（計算ロジック）
- 固定費自動計上:
  - 月初アクセス時に当月分を自動計上
  - `kakebo_last_fixed_applied` で二重計上を防止
- 予算進捗:
  - 80%超: 注意（アンバー）
  - 100%超: 超過（レッド）
- 今日使える金額:
  - 基本式: 残り予算 ÷ 残り日数
- カード引落:
  - 締め日・支払日・繰延月オフセットから引落日を算出
  - 必要に応じて `actualWithdrawalDate` を優先

## 7. 外部連携
- Anthropic API（レシートOCR）
  - 用途: 画像から日付・店名・金額を抽出
  - 呼出先: `https://api.anthropic.com/v1/messages`
  - モデル: `claude-sonnet-4-20250514`
  - 注意: ブラウザ直接呼出のため、APIキー管理に注意
- CSV取込（Papa Parse）
  - 対応: 汎用列検出 + 特定フォーマット（実装依存）
  - 文字コード: Shift_JIS フォールバックあり

## 8. 非機能要件
- 性能:
  - 単一ユーザー・端末内利用を前提
- 可用性:
  - ネットワーク不要（OCR利用時を除く）
- セキュリティ:
  - APIキーは `localStorage` 保持のため共有端末利用を非推奨
- 保守性:
  - `src/types`, `src/lib`, `src/store` を責務分離

## 9. ディレクトリ責務
- `src/pages`: 画面単位のUI/ユースケース
- `src/components`: 共通UI
- `src/store`: グローバル状態管理
- `src/lib`: 純ロジック（計算、CSV、保存、AI連携）
- `src/types`: 型定義
- `.github/workflows/deploy.yml`: 自動デプロイ定義

## 10. 実行・ビルド・デプロイ
- ローカル開発:
  - `npm install`
  - `npm run dev`
- ビルド:
  - `npm run build`
- プレビュー:
  - `npm run preview`
- 本番公開:
  - `main` へ push で GitHub Actions が Pages に自動デプロイ
